## 생성자 함수

### 객체 리터럴

```jsx
let user = {
	name : 'Mike',
	age : 30,
}
```

### 생성자 함수

```jsx
//생성자 함수는 첫 글자를 대문자로 하는 것이 관례!
function  User(name, age){
	this.name = name;
	this.age = age
}

let user1 = new User('Mike',30);
let user2 = new User('Jane', 22);
let user3 - new User('Tom', 17);
```

### ⭐️생성자 함수 로직

A와 같이 this = {} , return this; 코드가 없어도 new를 사용한다면 실제로 동작할 땐 B처럼 동작함

```jsx
A
function User(name, age){
	this.name = name;
	this.age = age;
}
new 함수명(); 

------------------------------------------------------------

B
function User(name, age){
	this = {}  // 빈 객체를 만듬 this에 할당 
	this.name = name;  //this 프로퍼티에 추가
	this.age = age;
	return this;  // 객체를 반환  
}
new 함수명();  
```

→ 생성자 함수라고 특별한 점이 없다. 어떠한 함수라도 new를 붙여 사용하면 이 알고리즘과 동일하게  동작한다.

### 상품 객체를 생성해보자.

```jsx
function Item(title, price){
// this = { }
this.title = title;
this.price = price;
this.showPrice = function() {
	console.log(`가격은 ${price}원 입니다.`)
}
// return this;
}

const item1 = new Item("인형", 3000);
const item2 = Item("가방", 4000);  //new 를 안붙히면 그냥 함수가 실행됨
const item3 = new Item("지갑", 9000);

console.log(item1, item2, item3);
// Item { title: '인형', price: 3000, showPrice: [Function (anonymous)] } 
// undefine
// Item { title: '지갑', price: 9000, showPrice: [Function (anonymous)] }
item3.showPrice();
// 가격은 9000원 입니다.
```
